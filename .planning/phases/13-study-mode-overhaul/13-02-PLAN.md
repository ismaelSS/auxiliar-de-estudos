c) Add new private method `handleCustomStudy()`:

  ```java
  private void handleCustomStudy() {
      ReviewDashboardView dashboard = view.getDashboard();
      if (dashboard == null || !dashboard.isCustomStudyMode()) return;

      List<String> selectedNames = dashboard.getSelectedThemeNames();
      if (selectedNames.isEmpty()) return;

      // Show question count dialog
      int count = askQuestionCount();
      if (count == 0) return; // cancelled

      // Resolve theme names to Theme objects
      List<Theme> selectedThemes = new ArrayList<>();
      for (Theme theme : themes) {
          if (selectedNames.contains(theme.getName())) {
              selectedThemes.add(theme);
          }
      }

      if (selectedThemes.isEmpty()) return;

      RoundState roundState = RoundState.createCustomStudyRound(selectedThemes, count, statsService);
      if (roundState.getSelectedQuestionsCount() == 0) return;

      StudyRoundView studyRoundView = new StudyRoundView();
      StudyRoundController studyRoundController = new StudyRoundController(
              roundState, studyRoundView, screenController, statsService);
      studyRoundController.setOnRoundEndCallback(() -> {
          refreshDashboard();
          screenController.switchTo("themeSelection");
      });
      studyRoundController.initialize();
  }
  ```

  This method:
  1. Gets selected theme names from the dashboard
  2. Shows the question count dialog (already created in Plan 1 Task 1)
  3. Resolves names to Theme objects from the loaded themes list
  4. Creates a custom study round via RoundState.createCustomStudyRound()
  5. Starts the round using the same StudyRoundController flow as regular review
  6. On round end, refreshes the dashboard and returns to theme selection

  NOTE: The SM-2 recording happens automatically — StudyRoundController.handleExit() calls statsService.recordRound(results) for ALL rounds, regardless of how they were created. The RoundResult objects contain the theme name from each question, so SM-2 updates are applied correctly per theme.

d) No changes needed to askQuestionCount() — it was already added in Plan 1 Task 1 and works for both single-theme and multi-theme flows.
