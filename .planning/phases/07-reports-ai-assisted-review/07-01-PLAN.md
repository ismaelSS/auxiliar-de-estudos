  <action>
Add two new public methods to StatsService.java, after the existing getLowestScoreQuestions() method (after line 154).

**Method 1 — getDominio(String themeName):**
- Return type: String
- Get ThemeStats from data.getThemes().get(themeName)
- If ts is null OR ts.getQuestions().isEmpty(), return "N/A"
- Count questions where qs.getScore() > 0 (positive score means the user answered correctly at least once more than incorrectly; zero means never answered or balanced)
- Calculate percentage: (positiveCount * 100) / ts.getQuestions().size() — use integer division
- Return String.valueOf(percentage)
- Per D-01/DOM-01: "Domínio" = percentage of questions with positive score (score > 0, NOT score >= 0)

**Method 2 — getLowestScoreQuestionsByTheme(String themeName, int limit):**
- Return type: List<Map.Entry<String, Integer>>
- Get ThemeStats from data.getThemes().get(themeName)
- If ts is null, return empty ArrayList
- Iterate ts.getQuestions().entrySet(), building List<Map.Entry<String, Integer>> with each entry's key (question ID string) and value (questionScore.getScore())
- Sort by value ascending (lowest score first) using Map.Entry.comparingByValue()
- Return subList(0, Math.min(limit, list.size()))
- This is the per-theme variant of the existing getLowestScoreQuestions(int limit) at line 141 — same pattern but filtered to one theme

Do NOT modify any existing methods. Do NOT change the existing getAproveitamento() or getLowestScoreQuestions() signatures.
