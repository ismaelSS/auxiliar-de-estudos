<objective>
Establish the question identity layer — every question gets a unique `id` string, the data model carries it end-to-end, the theme JSON files are annotated, and all matching/sorting logic switches from question text to question ID.

**Purpose:** Enable stable cross-referencing between stats data and questions regardless of question text edits. Provide the ID field needed by Waves 2 and 3 for score-based stats and aproveitamento display.

**Output:**
- Modified `Question.java` — new `id: String` field
- Modified `RoundResult.java` — new `questionId: String` field
- Modified `ThemeLoader.java` — duplicate ID validation
- Modified `RoundState.java` — ID-based matching in `createReinforcementRound()`
- Updated `themes/java-record.json` — 30 questions with IDs
- Updated `themes/java-modificadores.json` — 25 questions with IDs
</objective>
