# Phase 12 — UI Design Contract

> Visual and interaction contract for Spaced Repetition & Visual Review Dashboard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (JavaFX programmatic views, no FXML) |
| Preset | not applicable |
| Component library | none (JavaFX 25 built-in controls) |
| Icon library | none (text symbols: ⏰, ●, █) |
| Font | System default (`-fx-font-family: "System"`) — inherited from theme.css `.root` |

**CSS source:** `src/main/resources/styles/theme.css` — all new classes must follow existing naming conventions (`.surface`, `.button-primary`, `.label`, etc.)

---

## Spacing Scale

Declared values (multiples of 4, matching JavaFX Insets usage):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Dot/line gaps in timeline, inline badge padding |
| sm | 8px | Compact element spacing (timeline row gap, button internal padding) |
| md | 16px | Default element spacing (card inner padding, section gaps) |
| lg | 24px | Section padding (dashboard top/bottom margins) |
| xl | 32px | Layout gaps (between card list and timeline) |
| 2xl | 48px | Major section breaks (between dashboard header and cards) |
| 3xl | 64px | Page-level spacing (dashboard outer padding) |

Exceptions: none

---

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (1em — inherits from root zoom) | 400 (normal) | 1.5 |
| Label | 12px (0.85em) | 400 (normal) | 1.3 |
| Heading | 20px (1.4em) | 700 (bold) | 1.2 |
| Badge | 12px (0.85em) | 600 (semibold) | 1.0 |

Notes:
- All sizes are relative (`em`) — they inherit from the root `-fx-font-size` set by Ctrl+scroll zoom (Phase 8)
- Body 14px = base; Heading 20px = `.title` class (already 1.4em in theme.css)
- 3 font weights used: 400 (normal), 600 (semibold — badge only), 700 (bold) — matching existing `.label` and `.title` patterns

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#020817` | Background, main surfaces, dashboard root — matches `.background` |
| Secondary (30%) | `#1a1a2e` | Theme cards, search field, timeline backgrounds — matches `.surface` |
| Accent (10%) | `#fe9a00` | Action buttons (Revisar), timeline dots, search focus border, overdue count badge background |
| Destructive | `#e74c3c` | Overdue priority indicator, Aprendendo phase badge |

Accent reserved for:
- "Revisar" button background (primary CTA)
- Timeline dot fill
- Search field focus border
- "⏰ N" overdue badge background
- Selected tab indicator (existing TabPane pattern)

**Priority colors (card left border):**
- `#e74c3c` — Overdue (nextReview < now)
- `#fe9a00` — Due today (nextReview == today)
- `#27ae60` — No immediate review (nextReview > today or never reviewed)

**Fixation phase badge colors:**
- `#e74c3c` — Aprendendo (red)
- `#fe9a00` — Revisão (orange)
- `#27ae60` — Fixa (green)
- `#1abc9c` — Domínio (teal)

**Fixation bar segment colors:**
- `.fixation-segment-aprendendo` → `#e74c3c`
- `.fixation-segment-revisao` → `#fe9a00`
- `.fixation-segment-fixa` → `#27ae60`
- `.fixation-segment-dominio` → `#1abc9c`

## Interaction Contract

| Trigger | Action | Feedback |
|---------|--------|----------|
| User types in search field | Filter theme cards by name (real-time) | Non-matching cards hidden instantly |
| User clicks "Revisar" | Create due-only round → switch to StudyRoundView | Card button pressed state |
| User clicks "Marcar como feita" | Confirmation dialog → SM-2 quality=5 for overdue → refresh dashboard | Dialog appears; on confirm: card updates |
| Round completes | Auto-record SM-2 results → return to dashboard | Dashboard refreshed with new priority orders |
| Window resized | Cards reflow, search and timeline remain scrollable | No horizontal overflow |
| Ctrl+scroll | Root font-size adjusts (inherited from Phase 8) | All text scales proportionally |
