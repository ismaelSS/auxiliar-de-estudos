       - Current code loses theme association by collecting questions into a flat list first, then shuffling
       - Replace with: iterate themes, for each question in the theme, create a RoundQuestion that includes the theme name
       - Then shuffle the `roundQuestions` list (not the intermediate `all` list)
       
       New logic:
       ```java
       roundQuestions = new ArrayList<>();
       for (Theme theme : themes) {
           List<Question> themeQuestions = theme.getQuestions();
           int take = Math.min(questionsPerTheme, themeQuestions.size());
           for (int i = 0; i < take; i++) {
               Question q = themeQuestions.get(i);
               List<String> original = q.getOptions();
               String correctText = original.get(q.getCorrect());
               List<String> shuffled = new ArrayList<>(original);
               Collections.shuffle(shuffled);
               int newCorrect = shuffled.indexOf(correctText);
               roundQuestions.add(new RoundQuestion(theme.getName(), q, shuffled, newCorrect));
           }
       }
       Collections.shuffle(roundQuestions);
       ```
       
       Remove the old `List<Question> all = new ArrayList<>()` variable and the old loop that iterated `all`.

    3. **Add new public method:**
       ```java
       public String getCurrentThemeName() {
           return roundQuestions.get(currentIndex).themeName;
       }
       ```

    No changes needed to other existing methods (getCurrentQuestion, getCurrentOptions, getCurrentCorrectIndex, checkAnswer, advanceToNext, etc.) — their implementation stays identical.

    Maintain the existing `correctCount`, `totalAnswered`, `currentIndex` tracking — no changes to stats tracking in RoundState (that stays in StudyRoundController via RoundResult accumulation).
