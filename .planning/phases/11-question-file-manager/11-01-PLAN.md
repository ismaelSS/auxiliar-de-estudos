# Plan 11-01: Tab system refactor

## Objective
Refactor ThemeSelectionView to use a TabPane, enabling the Gerenciar tab while embedding Reports as a tab instead of a separate screen.

## Implementation

### ThemeSelectionView
- Replace `VBox root` with `TabPane root`
- Create Tab `"Jogar"`: wraps existing VBox children into a VBox inside the tab
- Create Tab `"RelatĂłrios"`: placeholder with no content (lazy-inited)
- Create Tab `"Gerenciar"`: placeholder with no content (lazy-inited)
- Remove `relatoriosButton` field and getter
- Keep all existing getters for startButton, themeCheckboxes, etc.
- Expose tabs via `getJogarTab()`, `getRelatoriosTab()`, `getGerenciarTab()`

### ReportsView

### ReportsController
- Add `refreshAndShow(Tab tab)` â†’ clears old content, rebuilds, sets `tab.setContent(getContent())`
- Keep existing `initialize()` for backward compatibility

### ThemeSelectionController
- On "RelatĂłrios" tab selected â†’ lazy-init `ReportsController`, call `refreshAndShow(tab)`
- On "Gerenciar" tab selected â†’ lazy-init `QuestionFileManagerController`, populate tab
- Use `tabPane.getSelectionModel().selectedItemProperty()` listener

- No changes needed (ThemeSelectionController handles everything)
